// 02 — Case Studies

How I actually think

Decision logs, failures, constraints. Not just what I built — how and why.

Case Study 01 — Competitor AI
Building a product intelligence tool from scratch
Problem → Web Search → LLM Analysis → Report
The constraint: LLM outputs were inconsistent — sometimes too verbose, sometimes missing key comparisons. I had to engineer structured prompts with explicit JSON schema constraints, then build a validation layer before rendering.

What I'd change: Earlier testing with real PMs would have revealed the output format mattered more than the depth of analysis.
"Prompt engineering is product engineering. The UX of an LLM output is the prompt itself."
Case Study 02 — AI SWOT Analyzer
Connecting AI outputs to existing team workflows
Input → LLM → Sheets API → PDF
The constraint: Google Sheets API rate limits and auth scopes were the hardest part — not the AI. Learned that integration complexity compounds fast when chaining multiple external APIs with authentication.

What I'd change: Build the export pipeline first, then add the AI layer. Infrastructure before intelligence.
"The hardest part wasn't the AI — it was the plumbing. That's where most AI products actually fail."

// 04 — Build Log

Lab notes

Short notes on what I'm learning, building, and figuring out.

Feb 2025
Structured LLM outputs are a product decision, not just a technical one
While building Competitor AI, I found that asking an LLM for "a comparison" produced inconsistent lengths and formats. Enforcing JSON schema in the prompt — and validating before rendering — fixed 80% of the quality issues. The prompt is the product spec.
LLMs · Prompt Engineering
Jan 2025
API chaining is where AI projects actually break
Integrating Gemini API + Google Sheets API + PDF export in one pipeline taught me that error handling at each layer is non-negotiable. A silent failure in the Sheets auth broke the whole chain. Now I always log and validate at every API boundary.
Integration · Python · APIs
Dec 2024
Agentic AI is mostly about trust, not capability
At GDG Agentathon, the hardest design question wasn't "can the agent do this?" but "when should it ask for confirmation?" Users don't trust agents that act silently. Human-in-the-loop checkpoints are a feature, not a fallback.
Agentic AI · UX · GDG Hackathon
Nov 2024
Streamlit is a legitimate prototyping superpower
For the AQI Visualizer I went from idea to deployed app in two days with Streamlit. For internal tools and AI demos, it removes all the frontend friction and lets you test the actual product hypothesis. Not production-forever, but perfect for signal-finding.
Streamlit · Python · Deployment
