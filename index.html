// 02 — Case Studies

How I think through problems

The decisions, trade-offs, and lessons from building real AI products.

Competitor AI · Architecture decision
Why I chose structured JSON output over free-form LLM text
Problem → Inconsistent output → JSON schema → 80% quality fix
Asking an LLM for "a comparison" produced wildly inconsistent lengths, formats, and depth. Enforcing a JSON schema in the prompt — and validating the response before rendering — fixed the majority of quality issues. The prompt became the product spec.
"Structured LLM output is a product decision, not a technical one. The schema you enforce is the quality floor you set."
SWOT Analyzer · Integration
How a silent auth failure broke an entire three-API pipeline
Gemini API → Sheets auth fail → Chain broken → Log every boundary
Integrating Gemini API + Google Sheets API + PDF export in one pipeline taught me that silent failures are the most dangerous. A Sheets auth issue broke everything with no error. I now log and validate at every API boundary, always.
"In multi-API systems, a silent failure is worse than a loud crash. Always instrument boundaries."
GDG Agentathon · Design
Why human-in-the-loop is a product feature, not a fallback
Agent design → Silent actions → User distrust → Checkpoints
The hardest design question wasn't "can the agent do this?" — it was "when should it ask for confirmation?" Users don't trust agents that act silently. Human-in-the-loop checkpoints increased perceived reliability dramatically.
"Agentic AI is mostly about trust, not capability. Transparency is a feature."
AQI Visualizer · Shipping
Going from idea to deployed app in 48 hours with Streamlit
Idea → Streamlit scaffold → API integration → Deployed
Streamlit removes all frontend friction and lets you test the actual product hypothesis. For internal tools and AI demos, it's a legitimate prototyping superpower — not production-forever, but perfect for signal-finding.
"Ship first to find signal. Streamlit is the fastest path from LLM output to user feedback."
